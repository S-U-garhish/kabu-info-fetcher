# EDINET 開示書類一括取得 指示書

## 概要

CSVで指定された銘柄について、EDINETから2025年以降の指定書類を取得し、所定のディレクトリ構造に保存する。

---

## 入力

- **CSV形式**：1列目に証券コード（5桁）、2列目に企業名
- **例**：
  ```
  7203,トヨタ自動車
  6758,ソニーグループ
  ```

---

## 取得対象書類

| 書類種別 | EDINET書類種別コード | 取得形式 |
|---|---|---|
| 四半期報告書 | `140` | PDF・XBRL・CSV |
| 有価証券報告書 | `120` | PDF・XBRL・CSV |
| 株主総会招集通知（定時） | `030` | PDF のみ |
| 株主総会招集通知（臨時） | `040` | PDF のみ |

- **対象期間**：提出日 2025-01-01 以降

---

## 取得手順

### 1. EDINET APIで書類リストを取得

EDINET API v2（書類検索）を使用。

```
GET https://disclosure2.edinet-fsa.go.jp/api/v2/documents.json
  ?date=YYYY-MM-DD
  &type=2
```

`date` を1日ずつ 2025-01-01 〜 当日までループし、レスポンスの `results` から以下の条件でフィルタリング：

- `secCode`（証券コード4桁＋`0`）がCSVの証券コードと一致
- `docTypeCode` が取得対象の書類種別コードと一致

### 2. 書類ファイルを取得

書類ごとに以下のエンドポイントでダウンロード：

```
GET https://disclosure2.edinet-fsa.go.jp/api/v2/documents/{docID}
  ?type=1   # PDF
  ?type=5   # XBRL（ZIPで返却）
  ?type=4   # CSV（ZIPで返却）
```

株主総会招集通知は `type=1`（PDF）のみ取得。

### 3. ファイル保存

以下の階層に保存する：

```
<親ディレクトリ>/
└── <証券コード>_<企業名>/
    └── <年度><書類種別>/
        └── <対象文書ファイル名>
```

**年度**：提出日の西暦4桁（例：`2025`）  
**書類種別**：下表の略称を使用

| docTypeCode | 略称 |
|---|---|
| `120` | `有価証券報告書` |
| `140` | `四半期報告書` |
| `030` | `定時株主総会招集通知` |
| `040` | `臨時株主総会招集通知` |

**ファイル名**：APIレスポンスの `docDescription` または `docID` をベースに、拡張子（`.pdf` / `.xbrl` / `.csv`）を付与。ZIPの場合は展開して格納。

**例**：
```
./output/
└── 7203_トヨタ自動車/
    ├── 2025有価証券報告書/
    │   ├── 7203_有価証券報告書.pdf
    │   ├── 7203_有価証券報告書.xbrl
    │   └── 7203_有価証券報告書.csv
    ├── 2025四半期報告書/
    │   └── ...
    └── 2025定時株主総会招集通知/
        └── 7203_招集通知.pdf
```

---

## 注意事項

- EDINET APIは1日単位のリクエストのため、日付ループが必要
- APIリクエストは間隔を空ける（推奨：1秒以上）
- ZIP展開後の不要ファイル（`__MACOSX` 等）は削除
- 既にファイルが存在する場合はスキップ（べき等性を確保）
- APIキーが必要な場合は環境変数 `EDINET_API_KEY` から読み込む
